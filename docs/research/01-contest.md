# ① 大会ルール・審査・審査員・特典

> AIでアイディアを練るときの「制約とルール」のインプット。評価軸・賞・審査員・日程・提供リソースをここに集約。

- 最終更新: 2026-07-10
- 対象: 都知事杯オープンデータ・ハッカソン2026（第6回）
- 主な出典: [募集要項](https://odhackathon.metro.tokyo.lg.jp/recruitment/)／[公式トップ](https://odhackathon.metro.tokyo.lg.jp/)／[都プレス2026-06-12](https://www.metro.tokyo.lg.jp/information/press/2026/06/2026061207)／[公式ガイドブック(FAQ)](https://odh-tokyo2026.code4japan.org/)

## 要点（3行）

- **ソロ可・無料・全日程不参加でも作品提出＋First Stage動画のみで可**。「東京都のオープンデータ等の活用」が応募必須条件。民間データ・生成AI併用は歓迎。
- 審査は **First Stage（2分プレゼン動画等の書類審査）→ 24チームがFinal Stage（審査委員採点＋オーディエンス投票）** の二段構え。First Stage通過率は約18%（2025実績）＝事実上の最大ボトルネック。
- 賞は**全8賞**が評価5軸と1:1対応。**部門区分はなく単一競技**で審査側が賞を授与（「強い1作を作り、結果的に該当賞へ乗せる」発想が正しい）。賞金なし（特典＝実装支援・露出・オープンバッジ・東京ポイント500pt）。

## 審査5軸（公式定義・原文確認済み）

| 軸 | 公式定義 | 対応する賞 |
|---|---|---|
| **データ活用** | オープンデータや民間データ等を有効活用しているか | データ活用賞 |
| **アイデア力** | 課題解消・期待実現のアプローチが秀逸で「**サービスとして成立**」しているか | アイデア賞 |
| **技術力** | 技術難易度・技術要素（アーキテクチャやAI等）の合理性・完成度 | 技術賞 |
| **ソーシャルインパクト** | 都民のQOL向上に貢献する社会的インパクト＋「**実現可能性が高い**」こと | ソーシャルインパクト賞 |
| **サービスデザイン** | 都民目線の価値を「**持続的に**」提供できるか | サービスデザイン賞 |

- **配点・重みは非公開**（公式は定義のみ。均等想定で設計）。
- 8賞 = 都知事杯（最優秀・5軸総合）＋上記5賞＋学生賞（高校生〜大学生のみ構成、**社会人ソロは対象外**）＋オーディエンス賞（Final Stage視聴者投票）。
- 罠: どの軸も「動くサービス＋続く設計」を前提。**PoC止まり・実現性の薄い企画は全軸で減点**。「アイデア力」すら「サービスとして成立」まで要求。
- 正しい狙い方: **5軸すべて最低ラインをクリアした上で1軸だけ突出**させる（1軸だけ高くても First Stage の総合評価で落ちる）。スライドで5軸それぞれに証拠を1つずつ置く。

## 都の政策的関心（隠れた加点の源泉）

- **2050東京戦略**の柱: チルドレンファースト（子育て）／防災／豊かなシニアライフ（高齢化）／次世代モビリティ／自己実現。全体トーンは「徹底したDXで都民の**手取り時間**を増やす」。
- 審査は社会実装（GovTech東京による行政採用に向けた技術検証、11月〜翌3月の実装支援、Demo Day）まで見据える → 「行政がそのまま採れる軽量・低コスト構成」が刺さる。
