# 診断結果ページ再構成 + 区別カラーテーマ 設計書

- 日付: 2026-07-17
- 対象: `/result/[slug]/`（構成・配色・ロジック）、`/ward/[slug]/`（配色のみ）
- ステータス: ユーザー承認済み設計。実装計画は別途 writing-plans で作成する

## 1. 背景と課題

現行の診断結果ページには次の問題がある。

1. PC画面で結果カード（OGP画像）が小さい。カード幅が `min(420px, 100%)` 固定のため。
2. PC画面でも下部固定のXシェアバーが常時追従し、カード内のシェアボタンと二重で見苦しい。
3. 結果の区の「にてる度%」が、下部の相性ランキングの区の%より小さくなることがある。診断割り当ては校正済み（全区1〜10%配分、未校正距離順位5位以内）で結果の区が距離最小とは限らないのに、相性側は生距離の `similarityPercent` をそのまま表示しているため。
4. 情報量と縦スクロールが多く、診断結果のインパクトが途中で薄れる。レーダーチャートが2回登場する。配色（アイボリー・金・黒・複数ベージュ）と装飾（枠線・影・角丸）が散らばって見える。
5. 区キャラクターごとのイメージカラーがあるのに、ページ全体の配色に活かされていない。

## 2. 決定事項（ユーザー確認済み）

| 論点 | 決定 |
|---|---|
| にてる度の逆転 | 結果の%を基準（最大）とし、相性側を相対スケーリングして常に単調に表示。校正済み割り当ての仕組みは変えない |
| シェア追従バー | 640px以下（モバイル）のみ表示。PCでは非表示 |
| カード拡大 | カード幅を拡大し、縦に伸ばさず、OGP画像の上に結果（区名・にてる度%）をオーバーレイ表示 |
| 共有専用画像（%入り個人別画像） | 見送り。Xシェアは現状のテキスト（%・タイプ入り）+ URL先OGPで十分とする |
| 「3つの特徴」コンテンツ | AI生成の区別テキストを新規作成（23区×3行）。中立・前向き原則に従う |
| テーマ適用範囲 | `/result/[slug]/` と `/ward/[slug]/`。トップ・図鑑は現行の金/茶のまま |

## 3. 設計

### 3.1 区別カラーパレット基盤（新規）

純関数モジュール `src/lib/wardPalette.ts` を追加する。種は `src/hero/wards.ts` の髪色由来テーマカラー `color`（23区分あり）。

```
wardPalette(baseColor: string): WardPalette
```

導出トークン（値はすべて hex 文字列で返す）:

| トークン | 導出 | 用途 |
|---|---|---|
| `bg` | ほぼ黒のブラウン（現行 #171009 系）に区色を約12%混合 | ページ背景 |
| `card` | 明るいアイボリー（現行 #f7ecd4 系）に区色を約7%混合 | カード地 |
| `accent` | 区色の彩度・明度をアイボリー地の上で読める範囲にクランプ | にてる度%数字・バッジ・シェアボタン |
| `accentDark` | 区色を暗背景上で映える明度に引き上げ | 暗背景上の見出し・リンク・キッカー |
| `ink` | ダークブラウン固定（現行 #4a3418 系） | カード内本文 |

- 実装は hex→HSL 変換 + 混合 + 明度クランプの純TypeScript。外部依存なし。
- コントラスト保証を Vitest で全23区について検証する:
  - `accent` 上の白または `ink` 文字（シェアボタン）: WCAG 4.5:1 以上
  - `accent` の大きい文字（にてる度%）を `card` 上に置く場合: 3:1 以上
  - `accentDark` を `bg` 上に置く場合: 4.5:1 以上
  - 淡色（千代田 #cdd3e8、荒川 #e0cb96）と暗色（品川 #48628e、墨田 #4a5f9a）を明示的な回帰ケースにする
- 適用方法: ResultPage / WardPage / WardDetail のルート要素に CSS カスタムプロパティ（`--w-bg` `--w-card` `--w-accent` `--w-accent-dark` `--w-ink`）としてインライン注入する。CSS 側は現行の金/茶をフォールバック値に持つ `var()` 参照へ置き換える。トップ・図鑑のスタイルは変更しない。
- 配色の原則: 1ページで使う色は「背景（区色がかった黒茶）」「カード（区色がかったアイボリー）」「アクセント（区色）」「墨色」の4系統に絞る。キャラページでは金色アクセントを区色アクセントに置き換える。

### 3.2 結果ページの新構成（診断済みユーザー）

上から順に:

**1. 結果カード（ファーストビューで完結）**
- カード幅 `min(820px, 100%)`（診断なし時ヒーローと同幅）。
- OGP画像（`/og/{slug}.jpg`）の下端にグラデーションスクリムを重ね、画像上に「◯◯ちゃん」（小）+「にてる度 78%」（大・`accent` 色）をオーバーレイする。区名はアートに焼き込み済みだが、23区どの構図でも確実に読めるようオーバーレイにも小さく載せる（スクリーンリーダー対応を兼ねる）。
- 画像の直下に1行: キャッチコピー（`wardTheme().catch`）+ ハッシュタグ2個（一致軸2本の極ラベル由来）。
- その下に横並び（PCは1行、SPは縦積み）: 「𝕏 で結果をシェアする」（`accent` 色でページ内最も目立つボタン。黒ボタンは廃止）+「詳しく見る」（`/ward/{slug}/` への導線）。
- h1「あなたに一番似ているのは…」はカードの上に残す。
- 現行カードの縦積みテキスト（名前・%の大表示・タイプ行・一致バッジ列）は撤去し、オーバーレイとセクション2以降へ移す。

**2. あなたの特徴**
- 見出し「◯◯区ちゃんタイプの特徴」+ AI生成の特徴3行（新規データ、3.4参照）+ タイプ総評1文（`personaType().description`）。

**3. なぜこの結果になったか**
- レーダーチャート（区の軸×あなたの軸のオーバーレイ）。**ページ内で唯一のレーダー**。
- 一致軸のみ（`matchedAxes`、常に2本）のトラック比較。現行の5軸全表示はやめる。
- AI相性解説（`loadAffinityText`）。

**4. 詳細（アコーディオン）**
- `<details>`/`<summary>` ベース。JSは追加しない。
- ［＋ 性格を詳しく見る］= キャラクター設定理由（`loadCharacterRationale`）
- ［＋ 実はこんな街］= データカード（`matchedStats`）+ 政策（`pickPolicyForAxes`）+「もっと知る」リンク
- ［＋ ステータスを見る］= StatBar 横棒一覧（現行 OPEN DATA STATUS の中身からレーダーを除いたもの）

**5. 似ているキャラクター**
- 相性上位3件。SPは横スクロール、PCは3カラムグリッド。
- 表示%は 3.3 の相対スケーリング値。

**6. 最後のシェアCTA**
- 「この結果、誰かに似ていませんか？」+ シェアボタンのみ。結果カードの再掲はしない。

補足:
- 下部固定シェアバー（`.result-share-bar`）と `has-share-bar` の下部パディングは `@media (max-width: 640px)` 内へ移動（640pxは既存ブレークポイントの慣習）。
- 診断なし閲覧者（sessionStorageに該当セッションがない場合）は現行構成（ヒーロー+「あなたも診断する」）を維持し、新パレットのみ適用する。

### 3.3 にてる度の相対表示ロジック

`src/lib/matching.ts` に純関数を追加する。

```
compatibilityPercents(resultDistance: number, compatibleDistances: number[]): number[]
```

- 結果の区の%は従来通り `similarityPercent(resultDistance)`。Xシェア文言に入る%も変わらない。
- 相性側の生% `p_i = similarityPercent(d_i)`（`compatibleDistances` は距離昇順前提）。
- `max(p_i) >= 結果%` の場合のみ、全体を比例スケーリングして最大値を「結果% − 1」に収める（`q_i = floor(p_i × (結果% − 1) / p_1)`）。比例なので相性内の順序（1位≥2位≥3位）は保たれる。
- 逆転がない場合は生の値をそのまま返す。
- 校正済み割り当て（`diagnosis-assignments.json`）には一切触れない。

### 3.4 コンテンツ: 区別「特徴3行」

- 新規データ `src/data/ward-traits.json`: 区コード→特徴3行（各1文・短文）。
- 生成方法: 既存の相性文・キャラ設定文と同様にAIで起草する。各区の性格（`catch`、5軸の傾き、キャラプロフィール）に沿わせ、UIコピーの中立・前向き原則（地域スティグマ回避）を守る。
- ローダー `src/data/traits.ts` + Vitest（23区分そろっている・各3行・空文字なし）。

### 3.5 ハッシュタグ導出

- 一致軸（`selectMatchedAxes` の返す2本）とユーザーベクトルの極性から、`src/lib/personaType.ts` の `POLES` ラベル（#都会派 #のんびり派 など）を2個導出する純関数を `src/lib/personaType.ts` に追加する。
- 結果カードに `#ラベル` 形式で表示する。Xシェア文言は変更しない（既存ハッシュタグ2つは固定のまま）。

### 3.6 装飾の整理

- 角丸を統一する（カード類 14px / ボタン・バッジ 999px を目安）。
- 枠線は結果カードなど要所のみ残し、セクションカードの枠線は基本廃止する。
- 影は結果カードとシェアボタンだけに絞る。
- セクション間の余白を広げ、カード内の余白をやや狭める。
- 英語キッカー（SHINDAN RESULT / YOUR TYPE 等）は縮小して日本語見出しの補助に留めるか、削減する。

## 4. やらないこと（非スコープ）

- %入り個人別共有画像の生成（クライアント合成含む）
- 診断割り当てロジック・`diagnosis-assignments.json` の変更
- トップページ・図鑑への区別テーマ適用
- `/ward/[slug]/` の構成変更（配色適用のみ）
- OGP原本・`npm run build:og` パイプラインの変更

## 5. テストと検証

## 7. 受け入れ基準

1. PC幅で結果カードが約820px幅になり、OGP画像上に区名とにてる度%が読めるオーバーレイで表示される。
2. 641px以上で下部固定シェアバーが表示されない。640px以下では従来通り表示される。
3. どの診断結果でも、相性ランキングの表示%が結果の%以上にならない。
4. レーダーチャートはページ内に1つだけ。
5. 詳細情報（性格・街データ・ステータス）がアコーディオンに収まり、初期表示のページ長が現行より大幅に短い。
6. `/result/[slug]/` と `/ward/[slug]/` の背景・カード・アクセントが区ごとの色になり、全23区でコントラスト基準を満たす。
7. `npm test` と本番相当ビルドが通る。
