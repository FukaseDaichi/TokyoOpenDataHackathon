# 診断ページめくり体験 設計

日付: 2026-07-19 / 対象: `src/ui/Diagnosis.tsx`、`src/ui/pages/ResultPage.tsx`、`src/lib/`、`app/zukan.css`

## 課題

1. 質問の切り替えが瞬間的なDOM差し替えで、アニメーションがない。
2. 選択肢タップへのフィードバックがなく、回答の手応えがない。遷移前に連打すると二重回答できる潜在問題もある。
3. 10問目の回答と同時に `router.push` で結果ページへ直行し、性格診断の要である「ため→発表」の瞬間がない。
4. 結果ページでは「にてる度%」とカードが静的に表示されるだけで、発表の高揚感がない。

## 方針

検討した3案（23枚のカードの舞台 / 絵本のページめくり / ベクトルが育つ）から**絵本のページめくり**を採用。図鑑モーダルに実装済みの絵本言語（羊皮紙 `#f7ecd4→#eeddb8`・金縁・`rotateY` の表紙開閉・金の粒子）と視覚言語を揃え、ヒーロー「絵本をめくる」→診断→図鑑がサイト全体で同じ一冊として繋がるようにする。

## 決定事項

### 1. フェーズ状態機械（純ロジック）

- `src/lib/diagnosisFlow.ts` を新設し、遷移reducerを純関数として実装する: `asking → stamping → turning → asking …（10問目のみ）… → finale → done`。
- イベントは PICK / STAMP_DONE / TURN_DONE / FINALE_DONE / SKIP。`asking` 以外での PICK は無視する（連打・二重回答防止）。
- reduced-motion時に stamping / turning / finale を短絡して即遷移するためのフラグを持つ。
- タイミング定数（封蝋 約400ms、ページターン 約600ms、フィナーレ 約2s）も同モジュールで管理する。定数は体感テンポ優先で調整してよいが、1回答あたりの遷移合計（封蝋＋ターン）は約1秒を超えないことを目安とする。
- Vitestを実装より先に追加する（10問のhappy path / 連打無視 / SKIP / reduced-motion短絡）。

### 2. 質問ページ（羊皮紙）

- `.diagnosis` を絵本のページに見立てる: 羊皮紙グラデーション・金縁3px・角丸20px（`.ward-modal-card` と同じトークン）。
- ページ右端に「残りページの厚み」の重なり線を置き、進行に応じて減らす。
- ページ下部中央にノンブル「— 3 / 10 —」（`aria-hidden`）。上部の金のプログレスバーは維持して `role="progressbar"` と値属性を付与し、既存の「N / 10」テキスト表示はノンブルへ移す。
- 表紙オープニング: 診断開始時に表紙（実在する `public/book-cover.webp` を使用。図鑑モーダルと同じCSSフォールバックあり）が約0.7秒で開いて1問目が現れる。

### 3. 封蝋スタンプ（回答フィードバック）

- タップした選択肢の上に金の蝋印（CSS/インラインSVG・✓入り・直径約34px）を表示し、`scale(1.6)→1`＋微回転のバックアウト系イージングでスタンプインさせる（約350〜400ms）。
- 非選択側の選択肢は opacity 0.6 へフェード。stamping / turning 中は両ボタンを無効化する。

### 4. ページターン

- ページを2層レンダリングする（現在の質問が上層、次の質問が下層）。上層を `transform-origin: left center` の `rotateY(0→-105deg)` でめくる（約0.6秒、`wardModalCoverOpen` と同系統の cubic-bezier）。
- めくれ中のページに影のグラデーションを重ねて紙の質感を出す。アニメーション終了後に下層を昇格する。
- ターン完了後、新しい質問見出し（`tabIndex={-1}`）へフォーカスを移す。コンテナの `aria-live="polite"` は維持する。

### 5. フィナーレ（ため→発表）

- 10問目の封蝋後、装飾ページ（3〜4枚目安）が加速しながらパラパラめくれ、最終ページ「あなたに一番似ているのは…」で一呼吸置く（計約2秒）。その後 `router.push('/result/[slug]/')`。
- 10問目の PICK 時点で `bestDiagnosisMatch` を計算し、フィナーレ中に結果ルートのprefetchと `/og/{slug}.jpg` のpreloadを行う（演出がローディング時間を隠す）。
- 絵本領域のタップ / Enter / Space で即スキップして遷移できる。視覚ヒント「タップでスキップ」を添える。

### 6. 結果ページの入場

- `.result-card` に入場アニメーション（opacity ＋ `translateY(12px)→0` ＋ `scale(0.97)→1`、約0.5秒）を付ける。
- 「にてる度%」を0から実値までrAFでカウントアップする（約0.9秒・easeOut）。診断セッションがある場合のみ。
- 金・シャンパン系の紙吹雪（約20粒・CSSアニメーション・一度だけ・`pointer-events: none`）を結果カード周辺に噴出する。診断経由の閲覧時のみ表示し、シェアリンク直訪では出さない。

### 7. アクセシビリティ・フォールバック

- `prefers-reduced-motion: reduce` では表紙・封蝋アニメ・ページターン・パラパラ・カウントアップ・紙吹雪をすべて無効化し、即時表示＋即時遷移（現行同等の挙動）にする。既存のreduced-motion節へ追記する。
- 蝋印はアニメーションのみ無効化して表示自体は残し、選択の視覚フィードバックは保つ。
- matchMedia非対応環境は `.ward-modal-static` と同様の静的クラス方針に倣う。

### 8. スコープ外（非目標)

- 質問文・選択肢コピーの変更 / 音 / 診断中のキャラクター表示やライブ順位表示（区の中立性維持）/ WebGL・3D / 新規ライブラリ追加 / 新規画像アセット（蝋印・紙質感はCSS/SVGで描く）。

## 新規アセット

不要。既存の `public/book-cover.webp` とCSS/インラインSVGのみで実現する。
