# 系統名のわかりやすい命名（設計）

日付: 2026-07-20

## 目的

区詳細・シェアカードに表示する「系統2」等の数値ラベルは、エンドユーザーに意味が伝わらない。街の雰囲気を表す日本語名に置き換える。リスク台帳P2（k-means番号に説明可能な意味がない）の緩和も兼ねる。

## 方式（案A: 代表区コードで名前を引く）

- k-meansの数値ラベルに直接名前を紐づけない（番号は入力順依存で不安定）。
- 各系統の「代表区コード → 名前」のマップを定義し、クラスタリング結果から代表区が属するクラスタへ名前を割り当てる。
- 不変条件: 6つの代表区は互いに異なるクラスタに属する。崩れた場合（データ更新で所属が変動）は例外を投げてビルド/テストを落とし、命名の見直しを強制する。
- 現行の全区所属をスナップショットテストで固定し、意味上の回帰を検知可能にする（P2緩和）。
