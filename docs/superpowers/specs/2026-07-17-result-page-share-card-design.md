# 診断結果画面「ファーストビュー完結型 結果カード」設計

日付: 2026-07-17
対象: `src/ui/pages/ResultPage.tsx`、`src/ui/ShareCard.tsx` ほか結果画面まわり

## 目的と優先度

診断結果画面を改善する。優先度は次の順。

1. エンドユーザーが「共有したい！」と思い、実際にXで共有すること
2. 結果への納得感
3. 情報のわかりやすさ

前提: スマートフォン利用が多数派。共有経路はX（リンク共有）を最優先とする。

## 現状の課題

- シェアボタンがページ最下部に1つだけあり、感情のピーク（結果が出た瞬間）とCTAの位置が離れている。
- シェア文面が汎用的（区名+キャッチのみ）で、「自分だけの数字・言葉」が入っていない。
- ファーストビューに「にてる度%」「タイプ名」がなく、結果の要約が1画面で完結しない。

## 設計

### 1. シェア文面（xShareUrl の拡張）

Xの文字数は全角=2単位・半角=1単位で280単位、URLは常に23単位（t.co換算）。

| 要素 | 消費単位 |
|---|---|
| URL | 23 |
| `#うちの区ちゃん` | 15 |
| `#都知事杯オープンデータハッカソン` | 33 |
| 改行・スペース | 約6 |
| 本文に使える残り | 約203単位（全角約101文字） |

両ハッシュタグは必須で維持する（ハッカソン提出作品としての目的上、大会タグを外さない）。

本文テンプレート:

```
診断したら {区名}ちゃんと にてる度{XX}% だった！タイプは「{タイプ名}」らしい
#うちの区ちゃん
#都知事杯オープンデータハッカソン
{URL}
```

- `xShareUrl(ward, appUrl)` を拡張し、診断済みの場合は `percent`（にてる度）と `personaName`（タイプ名）を受け取ってパーソナライズ文面を生成する。未診断（シェアリンク閲覧者が再シェアするケースはないため対象外）は現行文面を維持する。
- タイプ名は `personaType()` の出力（最長「ファミリー志向×フレッシュ志向タイプ」= 36単位）。最悪ケース（区名3文字・最長タイプ名・にてる度100%）でも280単位以内に収まることをVitestで固定する。加重文字数を数えるヘルパーを純ロジックとして実装しテストする。

### 2. 結果ヒーローカード（ファーストビュー）

横長OGP画像（1200×630）のヒーロー表示をやめ、スマホ縦画面に収まるカードへ再構成する。

- 画像はキャラクター896px版（`public/` の配信用WebP）を使う。
- カード内の構成（上から）: 「あなたに一番似ているのは…」→ 区ちゃん名 → にてる度◯◯%（最大の視覚要素）→ タイプ名 → 一致軸バッジ（`selectMatchedAxes` の上位、最大2個。例:「しずけさが一致」）。
- 縁取り・配色は図鑑プレミアムカード（`wardTheme` のテーマカラー）を踏襲する。
- にてる度は既存の `similarityPercent(distance)` を自区（1位）の距離に適用して算出する。

未診断の閲覧者（シェアリンク流入）には%とタイプを出せないため、現行どおりキャラ+キャッチ表示とし、「あなたも診断する」を最優先CTAとして維持する（バイラルループの入口）。

### 3. シェアCTAの2箇所配置

- カード直下にXシェアボタンを置き、ファーストビュー内でシェア可能にする。
- 画面下部に追従バー（`position: fixed`）を追加し、Xシェアボタンを常時表示する。診断済みユーザーのみ表示する。
- ページ最下部の既存シェア枠は追従バーと重複するため削除する。
- 追従バーはコンテンツ最下部と重ならないよう `padding-bottom` で逃がす。`prefers-reduced-motion` 配慮は不要（静的固定表示）。

### 4. セクション構成の変更

タイプ名がカードへ移動するため、YOUR TYPE セクションはレーダー重ね描画+タイプ説明文に整理する。以降の順序は現行を維持する:

1. 結果ヒーローカード + シェアCTA
2. YOUR TYPE（レーダー+説明文）
3. WHY WE MATCH（軸トラック比較 — 納得感の中核、変更なし）
4. 実はこんな街（オープンデータフック、変更なし）
5. キャラクター設定理由（変更なし）
6. ステータス（変更なし）
7. 相性ランキング（変更なし）

### 5. テスト方針

- 先にTDD: シェア文面生成（パーソナライズ文面・加重文字数280単位以内・未診断時フォールバック）の純ロジックテストを `src/ui/` または `src/lib/` に追加する。
- `ResultPage.test.tsx` を新構成に合わせて更新する: カード内の%とタイプ名の表示、一致軸バッジ、追従バーの有無（診断済み/未診断）、最下部シェア枠の削除。

## 変更しないこと（YAGNI）

- LINE・Instagram向けのシェア導線、Web Share API、画像の動的生成（OGPはコード合成しない制約を維持）。
- 相性ランキング・街データ・ステータス各セクションの内容。
- 演出（リビール、カウントアップ）は今回のスコープ外。将来の追加余地として残す。

## 制約の確認

- 静的エクスポート（`output: 'export'`）のまま。実行時APIなし。
- 診断ベクトルと結果区コードは引き続き `sessionStorage` のみ。シェアURLに個人の診断結果は含めない（URLは区slugのみ）。
- UIコピーは日本語、区の表現は中立・前向き。
- 変更後は `npm test` と本番相当ビルドで検証し、レンダリング設計書（`docs/system-design/`）の該当箇所を更新する。
