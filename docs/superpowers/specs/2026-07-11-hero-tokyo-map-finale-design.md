# ヒーロー更新: 全23区SSR反映・東京3Dマップ集結・スマホ雛壇 設計

日付: 2026-07-11 / 承認済み

## 背景

- `assets/characters/ssr/` に23区すべてのSSR画像が揃った（従来は13区）
- 最終集結シーン（Scene4）は現在「星座」= 垂直平面に東京の地理を写した浮遊配置。
  ユーザー要望は「星座というより東京の3Dマップみたいな見せ方」
- スマホ縦画面では横圧縮だけで23枚が重なって判別しづらい → 「ミニキャラ集結」に
- `assets/title.png`（ロゴ）は透過版の用意待ち。透過PNGが来たら組み込む（本設計のスコープ外）

## 決定事項

### 1. 残り10区の実画像反映

- [wards.ts](../../../src/hero/wards.ts) の `slug: null` 10区（中野・杉並・豊島・北・荒川・板橋・練馬・足立・葛飾・江戸川）にローマ字スラッグを設定
- `node scripts/build-hero-images.mjs` で `public/characters/ssr/` に23区×{512,896}のwebpを再生成
- クローズアップ演出は現行6区のまま変更なし

### 2. 最終シーン「東京3Dマップ」（横長画面 aspect >= 1）

- 星座配置を廃止し、**地面プレーンの上に各カードが地理位置で立つジオラマ**へ
  - `geo.x` → ワールドx（東西）、`geo.y` → ワールドz（北ほど奥）
  - カード下端が地面に接地（y = 地面 + 1.5 × 集結スケール）
  - 集結スケールは全カード一律（マップでは 0.72 目安、視覚確認で調整）
- カメラは終盤、**斜め上から見下ろす俯瞰**へ引く（cameraPose にアスペクト比を渡し分岐）
- 隣接関係の星座ライン（EDGES）は**地面に描く金色ライン**に変更（道路網の見立て）
- 地面: 大きな暗色プレーン + 淡い金のグロー。集結フェーズでフェードイン
- 最終コピー「23区が、データの星座になった。」→「23区が、データの地図になった。」

### 3. スマホ縦画面の集結（aspect < 1）

- 地理マップの代わりに**集合写真風の雛壇**：5列×5段（最終段3枚は中央寄せ）
  - 後列ほど高く・奥に配置。カードはミニスケールで重なりゼロ
  - 区名ラベル・キャッチは非表示（labelOpacity=0）。タップでモーダルは従来どおり
  - 地面ラインは非表示
- 雛壇座標は manifest で決定的に事前計算し、timeline がアスペクト比で選択

### 4. アーキテクチャ

- `manifest.ts`: `constellation` → `mapPos` / `podiumPos`（+ 各スケール）へ置き換え。
  `constellationDelay` → `gatherDelay` に改名
- `timeline.ts`: `scenePhases.constellation` → `gather` に改名。
  `cameraPose(t, aspect)` / `cardPose(card, t, aspect)` が最終シーンをアスペクトで分岐
- `HeroCanvas.tsx` / `HeroOverlay.tsx`: 描画反映のみ（従来方針を維持）
- 純ロジック（座標・カメラ・フェーズ）はTDD。テストは
  「マップで接地・地理順序保存」「雛壇で重なりなし・画面内」「縦横の分岐」を検証
