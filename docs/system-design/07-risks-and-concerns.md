# 現行コードの懸念事項

この文書は、現行実装を正としてコード、設定、データ、テスト、生成物を照合した時点のリスクと保守上の注意をまとめる。将来機能の一覧ではなく、現在の正しさ、再現性、運用性に影響する事項だけを対象とする。

## 優先度の定義

- **P1**: 公開内容の誤り、配信データの不整合、提出・再配布時の権利確認不能につながるため先に解消したい
- **P2**: 現在は主要導線が動くが、更新、配信、利用環境、アクセシビリティ、性能、セキュリティの面で問題化しやすい
- **P3**: 保守性と検証品質の改善として有効だが、直ちに公開機能を壊さない

## 受容している制約

次は欠陥ではなく、静的・無料・個人情報を保持しない構成として意図的に受容している。

- 共有URLから回答内容や相性ランキングを復元しない。
- データ更新には再集計と再デプロイが必要。
- 外部API停止の影響を受けない代わりに、表示値は取得時点のスナップショットである。
- WebGL不可時は演出品質を下げ、診断と図鑑の機能を優先する。
- three.js本体は `HeroCanvas`（トップ）と `WardMap3D`（区詳細）の動的チャンクへ重複して含まれる。2026-07-19にwebpack splitChunksで共有チャンク `three-vendor` への統合を試したが、チャンクが両者のunion（832KB）になり単一ページ訪問のLighthouse計測が悪化した（トップ94→89、区詳細73→63・転送+359KB）ためリバートした。ページ単位でtree-shakeされた重複の方が初回訪問体験で優位という計測結果に基づき、重複を受容する。
